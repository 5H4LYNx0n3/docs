<TopBlock>

By default, the result of a query that returns one or more records (such as a `User`) includes the following:

* **All** scalar fields defined in the Prisma schema
* **Zero** relations, such as a user's posts or profile

This is the **default selection set**. The following example returns the default selection set:

<CodeWithResult expanded="{true}">
<Cmd>

```ts file=script.ts
// Query returns User or null
const getUser: User | null = await prisma.user.findOne({
  where: {
    id: 22,
  },
})
```

</Cmd>
<CmdResult>

```js
{
  id: 1,
  name: "Alice",
  email: "alice@prisma.io",
  profileViews: 0,
  role: "ADMIN",
  coinflips: [true, false],
}
```

</CmdResult>
</CodeWithResult>

To return specific fields or include relations, use `select` <span class="api"></span> and `include` <span class="api"></span> respectively.

</TopBlock>

## Select specific fields


Use `select` <span class="api"></span> to return a specific subset of fields. The following example returns the `email` and `name` fields only:

<CodeWithResult expanded="{true}">
<Cmd>

```ts file=script.ts
// Returns an object or null
const getUser: object | null = await prisma.user.findOne({
  where: {
    id: 22,
  },
  select: {
    email: true,
    name: true,
  },
})
```

</Cmd>
<CmdResult>

```js
{
  name: "Alice",
  email: "alice@prisma.io",
}
```

</CmdResult>
</CodeWithResult>

## Include relations

To included related records, such as a user's posts, use the `include` <span class="api"></span> option. The following example returns a single user _and_ that user's posts:

<CodeWithResult>
<Cmd>

```ts file=script.ts
const getUser = await prisma.user.findOne({
  where: {
    id: 19,
  },
|  include: {
|    posts: true,
|  },
})
```

</Cmd>
<CmdResult>

```js
{
  id: 19,
  name: null,
  email: 'emma@prisma.io',
  profileViews: 0,
  role: 'USER',
  coinflips: [],
  posts: [
    {
      id: 20,
      title: 'My first post',
      published: true,
      authorId: 19,
      comments: null,
      views: 0,
      likes: 0
    },
    {
      id: 21,
      title: 'How to make cookies',
      published: true,
      authorId: 19,
      comments: null,
      views: 0,
      likes: 0
    }
  ]
}
```

</CmdResult>
</CodeWithResult>

The following example returns all posts with the word `cookie` in the `title` field _and_ the author of each post.

<CodeWithResult>
<Cmd>

```ts file=script.ts
const getPosts = await prisma.post.findMany({
  where: {
    title: {
      contains: 'cookies',
    },
  },
|   include: {
|    author: true,
|  },
})
```

</Cmd>

Note that you **cannot** use `select` <span class="api"></span> and `include` <span class="api"></span> on the same level:



<CodeWithResult>
<Cmd>

```ts file=script.ts
// The follwing query returns an exception
const getUser = await prisma.user.findOne({
  where: {
    id: 19,
  },
|  select: { // No!
    email:  true
  }
|   include: { // No!
    posts: {
      select: {
        title: true
      }
    }
  },
})
```

</Cmd>
<CmdResult>

```
Invalid `prisma.user.findOne()` invocation:

{
  where: {
    id: 19
  },
  select: {
  ~~~~~~
    email: true
  },
  include: {
  ~~~~~~~
    posts: {
      select: {
        title: true
      }
    }
  }
}


Please either use `include` or `select`, but not both at the same time.
```

</CmdResult>
</CodeWithResult>

### Select relation fields

<CodeWithResult>
<Cmd>

```ts file=script.ts
const getUser = await prisma.user.findOne({
  where: {
    id: 19,
  },
  include: {
    posts: {
      select: {
        title: true
      }
    }
  },
})
```

</Cmd>
<CmdResult>

```js
{
  id: 19,
  name: null,
  email: 'emma@prisma.io',
  profileViews: 0,
  role: 'USER',
  coinflips: [],
  posts: [ { title: 'My first post' }, { title: 'How to make cookies' } ]
}
```

</CmdResult>
</CodeWithResult>
