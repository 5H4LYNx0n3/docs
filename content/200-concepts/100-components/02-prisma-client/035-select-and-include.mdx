<TopBlock>

By default, when a query returns records (as opposed to a count), the result includes the **default selection set**:

- **All** scalar fields defined in the Prisma schema
- **None** of the relations

To customize the result:

- Use `select` <span class="api"></span> to return specific fields
- Use `include` <span class="api"></span> to explicitly include relations

Selecting only the fields and relations that you require rather than relying on the default selection set can ✔ reduce the size of the response and ✔ improve query speed.

</TopBlock>

## Example schema

All examples are based on the following schema:

<details><summary>Expand for sample schema</summary>

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model ExtendedProfile {
  id        Int    @id @default(autoincrement())
  biography String
  user      User   @relation(fields: userId, references: id)
  userId    Int
}

model User {
  id           Int             @id @default(autoincrement())
  name         String?
  email        String          @unique
  profileViews Int             @default(0)
  role         Role            @default(USER)
  coinflips    Boolean[]
  posts        Post[]
  profile      ExtendedProfile
}

model Post {
  id         Int        @id @default(autoincrement())
  title      String
  published  Boolean    @default(true)
  author     User       @relation(fields: [authorId], references: [id])
  authorId   Int
  comments   Json?
  views      Int        @default(0)
  likes      Int        @default(0)
  categories Category[]
}

model Category {
  id    Int    @id @default(autoincrement())
  name  String @unique
  posts Post[]
}

enum Role {
  USER
  ADMIN
}
```

</details>

## Return the default selection set

The following query returns the default selection set (all scalar fields, no relations):

<CodeWithResult expanded="{true}">
<Cmd>

```ts file=script.ts
// Query returns User or null
const getUser: User | null = await prisma.user.findUnique({
  where: {
    id: 22,
  },
})
```

</Cmd>
<CmdResult>

```js
{
  id: 1,
  name: "Alice",
  email: "alice@prisma.io",
  profileViews: 0,
  role: "ADMIN",
  coinflips: [true, false],
}
```

</CmdResult>
</CodeWithResult>

## Select specific fields

Use `select` <span class="api"></span> to return a limited subset of fields instead of all fields.

The following example returns the `email` and `name` fields only:

<CodeWithResult expanded="{true}">
<Cmd>

```ts file=script.ts
// Returns an object or null
const getUser: object | null = await prisma.user.findUnique({
  where: {
    id: 22,
  },
  select: {
    email: true,
    name: true,
  },
})
```

</Cmd>
<CmdResult>

```js
{
  name: "Alice",
  email: "alice@prisma.io",
}
```

</CmdResult>
</CodeWithResult>

## Include relations

Use `include` <span class="api"></span> to include related records, such as a user's posts or profile, in the response.

The following example returns a single user _and_ that user's posts:

<CodeWithResult>
<Cmd>

```ts file=script.ts
const getUser = await prisma.user.findUnique({
  where: {
    id: 19,
  },
|  include: {
|    posts: true,
|  },
})
```

</Cmd>
<CmdResult>

```js
{
  id: 19,
  name: null,
  email: 'emma@prisma.io',
  profileViews: 0,
  role: 'USER',
  coinflips: [],
  posts: [
    {
      id: 20,
      title: 'My first post',
      published: true,
      authorId: 19,
      comments: null,
      views: 0,
      likes: 0
    },
    {
      id: 21,
      title: 'How to make cookies',
      published: true,
      authorId: 19,
      comments: null,
      views: 0,
      likes: 0
    }
  ]
}
```

</CmdResult>
</CodeWithResult>

The following example returns all posts with the word `cookie` in the `title` field _and_ the author of each post.

<CodeWithResult>

```ts file=script.ts
const getPosts = await prisma.post.findMany({
  where: {
    title: {
      contains: 'cookies',
    },
  },
|   include: {
|    author: true,
|  },
})
```

</Cmd>

Note that you **cannot** use `select` <span class="api"></span> and `include` <span class="api"></span> _on the same level_:

<CodeWithResult>
<Cmd>

```ts file=script.ts
// The follwing query returns an exception
const getUser = await prisma.user.findUnique({
  where: {
    id: 19,
  },
|  select: { // No!
    email:  true
  }
|   include: { // No!
    posts: {
      select: {
        title: true
      }
    }
  },
})
```

</Cmd>
<CmdResult>

```
Invalid `prisma.user.findUnique()` invocation:

{
  where: {
    id: 19
  },
  select: {
  ~~~~~~
    email: true
  },
  include: {
  ~~~~~~~
    posts: {
      select: {
        title: true
      }
    }
  }
}


Please either use `include` or `select`, but not both at the same time.
```

</CmdResult>
</CodeWithResult>

To `select` specific record and related record fields, use a [nested `select`](#select-relation-fields).

#### Include a filtered list of relations

See [Working with relations](working-with-relations#filter-the-relations-list) to find out how to combine `include` and `where` for a filtered list of relations - for example, only include a user's published posts.

### Select specific relation fields

The following example selects the user's `name` and the `title` of each related post:

<CodeWithResult>
<Cmd>

```ts file=script.ts
const getUser = await prisma.user.findUnique({
  where: {
    id: 19,
  },
  select: {
    name: true,
    posts: {
      select: {
        title: true,
      },
    },
  },
})
```

</Cmd>
<CmdResult>

```js
{
  name: "Elsa",
  posts: [ { title: 'My first post' }, { title: 'How to make cookies' } ]
}
```

</CmdResult>
</CodeWithResult>

## Advanced examples

```ts file=script.ts
const getUser = await prisma.user.findMany({
  where: {
    posts: {
      some: {
        categories: {
          some: {
            name: {
              in: ['Food', 'Introductions'],
            },
          },
        },
      },
    },
  },
  select: {
    posts: {
      select: {
        categories: {
          select: {
            name: true,
          },
        },
      },
    },
  },
})
```
